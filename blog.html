Global Scope : If a variable is declared outside all functions or curly braces ({}), it exists in the global scope. The global variables can be accessed by any line of code in the program, including inside blocks.

Block Scope : If a variable is declared inside any curly braces({}).its exists in the block scope. let and const are block scoped and var is function scoped. We can not access variables declared with let and const inside a block, outside that particular block. but we can access variable declared with var keyword in this case.

What is event loop? Event loop continuously observes the call stack and call back queue . when it is empty it transfers the very first task to the call stack from the call back queue.

How it works?
1- Browser has superpowers that helps to the JS engine to execute some tasks, these superpowers include web API's such as console, location, DOM API(anything starts with document like document.get element ID), setTimeout (it's a timer in the browser), fetch()(it can fetch data from the remote server), and local storage(browsers can stores data locally)
2- Callback functions and event handlers are first stored in the Web API environment and then transferred to the callback queue.
3- Only the call back functions that come from Promises and mutation observers are stored in the Web API environment and then transferred to the microtask queue.
4- Event loop continuously observes the call stack and call back queue . when it is empty it transfers the very first task to the call stack from the call back queue.
3- Micro tasks are given priority over the tasks inside the callback queue by Event loop to send to the call stack.
